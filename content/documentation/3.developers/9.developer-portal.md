The **Developer Portal** is where you create and manage OAuth applications that connect to Coach Watts on behalf of athletes.

## Getting started

1. Sign in to Coach Watts
2. Go to **Settings → Developer** or visit [/developer](/developer)
3. Click **Create Application**
4. Fill in your app details and save

You receive a `client_id` and `client_secret`. Store the secret securely — it is shown only once at creation.

## Application settings

Each application has:

| Field              | Purpose                                                                                       |
| ------------------ | --------------------------------------------------------------------------------------------- |
| **Name & Logo**    | Shown on the OAuth consent screen                                                             |
| **Redirect URIs**  | Allowed callback URLs for the authorization flow                                              |
| **Scopes**         | Permissions your app requests (see [Scopes](/documentation/developers/scopes))                |
| **Webhook URL**    | Auto-generated incoming webhook endpoint (see [Webhooks](/documentation/developers/webhooks)) |
| **Webhook Secret** | Authenticates incoming webhook payloads                                                       |

## OAuth flow

1. Redirect users to the [authorization endpoint](/documentation/developers/authentication)
2. User approves scopes on the consent screen
3. Receive an authorization code at your redirect URI
4. Exchange the code for access and refresh tokens
5. Use the access token in `Authorization: Bearer` headers

## Testing your integration

- **Interactive API explorer**: [/\_docs/scalar](/_docs/scalar) (recommended) or [/\_docs/swagger](/_docs/swagger)
- **OpenAPI spec**: [/\_openapi.json](/_openapi.json)
- **Local development**: `http://localhost:3099/api/` with your dev credentials

## Rate limits

| Type                      | Limit                   |
| ------------------------- | ----------------------- |
| Unauthenticated (per IP)  | 100 requests / minute   |
| Authenticated (per token) | 1,000 requests / minute |

## MCP integration

To connect AI assistants instead of building a traditional app, see [MCP Server](/documentation/developers/mcp-server). MCP uses OAuth with dynamic client registration — no manual app setup required for tools like Cursor.
